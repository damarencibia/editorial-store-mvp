## Goal
Improve and extend the Bookstore MVP with profile management, admin CRM, registration UX, and various UI/UX fixes.

## Constraints & Preferences
- Reviews: 1 per user per product (UNIQUE INDEX + API + frontend enforcement)
- Any authenticated user can review; owner edits/deletes own; admin edits/deletes any
- Auth redirect must point to production URL, not localhost
- Account registration collects first name + last name
- Admin CRM needs customer freeze, role change, and account deletion (including auth.users via service role key)
- Cart delete button always visible on mobile
- Hero has single button scrolling to first store section (Top 10 del mes)

## Key Decisions
- Manual `attachProfiles` helper via batch fetch instead of PostgREST join syntax because FK `reviews.user_id` → `auth.users` (not `profiles.id`), so schema cache can't infer the relationship
- Created `AdminCustomersTable.vue` as standalone component instead of modifying shared `DataTable.vue` to avoid regression risk
- Delete customer uses `supabaseAdmin.auth.admin.deleteUser()` to remove from `auth.users` (service role key required)
- Name fields stored as combined `full_name` in `profiles` table, passed via Supabase `user_metadata` on signup
- Profile page uses Astro SSR (server-rendered sections: header, orders, reviews) with one Vue client component (`ProfileForm`) for name/password editing
- Reviews API endpoint (`/api/user/reviews`) uses server supabase client with `reviews, books!inner()` join (works because FK `book_id` is in public schema)
